在 Python 開發中，建立**虛擬環境（Virtual Environment）**是一項核心的最佳實踐。虛擬環境是一個獨立的資料夾，其中包含特定版本的 Python 直譯器以及一組相關的套件。

以下根據來源文件，說明虛擬環境的主要優點與必要性：

### 1. 解決套件版本衝突（核心必要性）
Python 應用程式通常需要標準函式庫之外的第三方套件，且不同專案可能依賴同一個套件的不同版本。
*   **版本衝突問題**：若應用程式 A 需要模組的 1.0 版，而應用程式 B 需要 2.0 版，直接安裝在系統環境中會導致衝突，使得其中一個專案無法運作。
*   **解決方案**：透過虛擬環境，應用程式 A 和 B 可以擁有各自獨立的環境，即使應用程式 B 將函式庫升級到 3.0 版，也不會影響到應用程式 A。

### 2. 保持系統環境的乾淨與穩定
*   **隔離性**：虛擬環境可以讓開發者在不影響作業系統內建 Python 環境的情況下，自由地安裝或測試套件。
*   **預設隔離**：虛擬環境預設會與系統中的套件隔離，僅能存取明確安裝在該環境中的內容。這能防止因安裝過多第三方套件而導致系統工具出錯。

### 3. 提升專案的可移植性與重現性
*   **環境一致性**：透過工具（如 Poetry 或 pip），開發者可以產生 `poetry.lock` 或 `requirements.txt` 檔案，確保所有開發者和部署環境都使用完全相同的套件版本。
*   **易於重建**：虛擬環境被視為**可拋棄的資源**，應能簡單地刪除並從頭重新建立。這意味著只要有依賴清單（如 `requirements.txt`），即可在任何機器上快速還原相同的開發環境。

### 4. 支援多樣化的開發場景
*   **單體倉庫與微服務**：對於包含多個服務（如後端與機器學習服務）的單體倉庫（Mono-repos），可以為每個資料夾分配不同的環境，避免環境過於臃腫或混亂。
*   **跨版本測試**：開發者可以輕鬆地在同一個系統上，針對 Python 3.10 和 3.12 等不同版本進行測試，只需為不同專案切換對應的環境即可。


總結來說，虛擬環境透過**環境隔離**解決了版本衝突，並藉由**精確的依賴管理**確保了專案在不同機器上的穩定運作，是現代 Python 開發不可或缺的工具。
